# Implementation Plan: Dev Auth Bypass and Patient Access Controls

**Branch**: `003-dev-auth-patient-access` | **Date**: 2026-04-04 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/003-dev-auth-patient-access/spec.md`

## Summary

Add a development-mode authentication bypass to `requireAuth` so developers
can call all endpoints locally without JWT tokens. Separately, document and
test the existing patient permission boundaries — code analysis confirms that
`assertPatientAccess` already enforces all required patient restrictions, so
US2 requires new tests only (no route or service changes).
